# 盈信官网多页面内容统一化实战记录（2026.5.22）

## 背景

盈信官网（ASP + Access 数据库）两个月前改过一次网站，但改得不彻底：
- 部分新增资质信息没加到首页的段落里（首页底部和关于盈信页面信息不对称）
- 资质名称表述不统一（写"示范基地"的地方没加括号补充"会计学会"）
- 协会会员写得太模糊（"省市行业协会会员"），应该明确写出具体协会名称

## 目标

将官网所有涉及资质的文字统一为最新的标准表述，并做加粗强调。

## 执行脚本

用 update.asp 分别修改两个 id，每个脚本走 RecordSet 读-改-写流程：

```asp
' == id=4 首页替换 ==
rs.Open "SELECT s_content FROM a_info WHERE id=4", conn, 1, 3
oldC = rs("s_content")
newC = Replace(oldC, _
  "公司系苏州工业园区会计服务外包示范基地成员单位，资质可查", _
  "公司系苏州工业园区会计服务外包示范基地（苏州工业园区会计学会）成员单位，资质可查")
newC = Replace(newC, _
  "苏州工业园区会计学会（服务外包示范基地）推荐成员、省市行业协会会员", _
  "苏州工业园区会计服务外包示范基地（苏州工业园区会计学会）推荐成员、苏州会计服务协会和江苏省代理记帐协会会员")
rs("s_content") = newC
rs.Update()
```

## 经验总结

### 下次遇到类似需求的操作流程

1. **先扫全表**：`SELECT id, s_name FROM a_info` 了解所有记录
2. **定位目标记录**：确认哪些 id 需要改
3. **逐条查看原始 s_content**：确认原文精确措辞（每一处都要看，不同记录中同一关键词的语境可能不同）
4. **写更新脚本**：每页独立一个脚本，分两次上传执行
5. **验证**：HTTP 抓取两个页面，搜索替换后的关键词确认
6. **清理**：删掉所有临时脚本

### 为什么不要用一条 SQL 批量干

两页的替换策略不同：
- 首页的"示范基地"前后有完整的句子语境（`公司系...成员单位`），做整句替换
- 关于盈信的"示范基地"只是部分名词，需要包裹 `<strong>` 标签加粗
- 两页的"协会会员"表述也不一样

一条 SQL 解决不了这种"相同关键词、不同语境、不同替换方式"的需求。

### FTP 上传要点

- 传含中文的 .asp 文件用 `storbinary()` 不是 `storlines()`
- 编码用 `latin-1`（Windows 中文服务器）
- ASP 文件头部加三件套：`Codepage=65001` + `Response.Charset=utf-8` + `Session.CodePage=65001`
