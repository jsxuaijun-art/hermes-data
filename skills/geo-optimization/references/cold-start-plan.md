# GEO 冷启动执行计划（财税行业版）

## 概述

适用于财税服务公司（代理记账、公司注册、税务合规）的 GEO 冷启动方案。按 P0/P1/P2 三级优先级组织，每项含具体操作和预估耗时。

---

## 🔴 P0 — 立即执行（1-5天）

| 事项 | 具体操作 | 预估耗时 |
|------|---------|---------|
| 官网加结构化数据 | 加 JSON-LD（Organization + LocalBusiness + FAQ 标记） | 1天 |
| 创建FAQ页面 | 官网加 20+ 个问答，覆盖 AI 常问关键词（如"苏州公司注册常见问题"） | 半天 |
| 统一信息口径 | 确保官网、公众号、抖音等所有渠道核心数据一致（更新年份、客户数等） | 半天 |

**原则：** P0 是地基——不改后面的事都不需要做。

---

## 🟡 P1 — 需要注册/启动（1-2周）

| 平台 | 注册什么 | 为什么 | 优先级 |
|------|---------|-------|-------|
| 百度百科 | 企业词条 | 百度系 T1 信源，文心一言直接采信 | ⭐⭐⭐ |
| 知乎 | 创始人/CEO 专业认证账号 | DeepSeek 偏好的 T2 信源，长文场景卡位 | ⭐⭐⭐ |
| 百家号 | 企业官方号 | 百度搜索收录最快的内容渠道 | ⭐⭐ |
| 天眼查/企查查 | 确保企业工商信息完整准确 | AI 交叉验证时会查阅，基础 T1 信源 | ⭐⭐ |

**原则：** 百度百科搞定文心一言，知乎搞定 DeepSeek，这两个是核心。

---

## 🟢 P2 — 可以同步做（2-4周）

| 平台 | 为什么 | 说明 |
|------|-------|------|
| 小红书账号 | 豆包（字节系）偏好抖音生态，小红书作补充 | 补充豆包信源密度 |
| 头条号 | 字节生态补充，与抖音内容联动分发 | 补充字节系信源 |
| 公众号 | 元宝偏好微信生态，现有内容可重新优化 | 元宝优先采信 |
| 行业垂类媒体 | 在会计网/中国会计视野等发专业文章 | 增强行业权威性 |

---

## 📋 执行检查清单

### 诊断阶段
- [ ] 在豆包/元宝/DeepSeek/文心搜索品牌核心关键词，记录提及率
- [ ] 搜索竞品关键词，记录 AI 排名
- [ ] 检查官网是否有结构化数据（用 Google Rich Results Test 或类似工具）

### P0 完成
- [ ] 官网 JSON-LD 结构化数据上线
- [ ] FAQ 页面发布（覆盖 20+ 常见问题）
- [ ] 所有渠道核心数据口径统一

### P1 完成
- [ ] 百度百科词条提交/更新
- [ ] 知乎专业认证账号注册
- [ ] 百家号企业号开通
- [ ] 天眼查/企查查工商信息核实

### P2 完成
- [ ] 小红书账号注册
- [ ] 头条号开通
- [ ] 公众号内容优化
- [ ] 行业媒体投稿（至少 1 篇）

---

## 💡 执行要点汇总

1. **数据口径统一是第一要务** — 经验年数、客户数、转介绍率、认证资质，所有渠道保持一致
2. **先 T1 后 T2** — 百度百科和官网搞定后，再去知乎、百家号等 T2 平台
3. **做完一项勾一项** — 检查清单是进度管理工具，也是对客户展示透明度的凭证
4. **内容可复用** — 知乎长文可以改写后发百家号，百家号内容可以压缩发小红书，不要从零做每一条
