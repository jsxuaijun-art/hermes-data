# 中小企业官网内容巡检清单

> 吸收自 `website-content-audit` skill。在动手修改数据库内容前，先用此清单做一次全面检查，避免修复了旧问题、忽略了新问题。
> 与 `db-website-content-mgmt` 配合使用：先审计 → 再修改。

---

## 核心理念

> 老板让你"看下网站"，不是让你设计新页面或搞 SEO，而是检查**现有的东西对不对**。

常见需求：换了地址但网站没更新、文案写错、资质过期、页脚/侧边栏内容和主内容打架。

**不要问为什么 — 直接动手查。**

---

## Step 1: 确定检查范围

听到"看一下网站"、"整体检查一下"、"官网浏览一下"等指令时，首先确认 URL。

- 通常用户自己的网站直接给 URL
- 先 curl 首页，了解整体结构
- 再按"关键页面"展开

---

## Step 2: 关键页面清单（中小企网站必查）

| 页面 | 检查重点 |
|------|---------|
| 首页（index / 根） | 公司简介文案、服务条数、年份 |
| 关于我们（about） | 创始人介绍、从业经验年数、公司资质陈述 |
| 联系我们（contact） | **地址**（最重要 — 最容易改完忘更新网站）、电话、微信二维码 |
| 服务页（service） | 服务项目清单、价格（如有） |
| 常见问题（FAQ） | 答案是否过时 |

---

## Step 3: 多层级一致校验

每个页面检查**三层结构**的内容一致性：

```
[1] 主内容区（正文）
    ↓ 对比 ↓
[2] 侧边栏/导航栏
    ↓ 对比 ↓
[3] 底部 footer
```

**最容易出问题的地方：地址。** 搬了办公室只改了一处，其他位置没同步。

---

## Step 4: 基础卫生检查清单

按优先级排列：

| 优先级 | 检查项 | 判断方法 |
|--------|--------|---------|
| 🔴 高 | 电话/地址是否正确 | 和用户确认的最新信息对比 |
| 🔴 高 | 公司名称/成立年份 | about.html 正文 vs footer |
| 🟡 中 | 文案是否 outdated（"今年"、"去年"等时间词） | 检查时间类表述 |
| 🟡 中 | SSL 证书 | URL 是否 https，Chrome 检查 |
| 🟡 中 | 链接有效性 | 导航链接、资质图片、二维码路径 |
| 🟢 低 | 图片文件名规范 | 路径是否统一（避免混合 /manage/ 和 /sz@yxmanage/） |
| 🟢 低 | 文字排版（字体、间距混乱） | 目测一致性 |
| 🟢 低 | 非点击链接（纯文本的 URL） | 检查链接是否可点开 |

---

## Step 5: 交付结构

直接给报告，结构固定：

```
一、首页/核心文案 ✅ 已确认 OK
   - 具体位置文案内容确认

二、联系我们 — 地址检查 ⚠️
   - 三层结构对比结果
   - ✅ 左栏正文正确
   - ❌ 侧边栏/页脚未同步（如有）

三、其他发现的问题
   （用简单表格，2列：问题 | 位置）

四、你现在需要我帮你做什么？
   - 给用户选择：修不修、怎么修、还是确认完事
```

**关键原则：**
- 不输出长分析文字，直接给检查结论
- 每项检查要有具体位置（文件路径或页面区域）
- 修之前先问用户要不要修 — 有些"过时"内容可能是故意留着
- 不要主动帮用户改服务器上的文件，除非用户明确说"改"

## 技术执行参考

```bash
# 获取页面内容
curl -sL --max-time 10 -A "Mozilla/5.0" <URL> 2>&1
```

### 复用技巧
- 核心页面：index / about / contact / service / FAQ
- 侧边栏和 footer 通常在同一个 HTML 文件里，反复出现
- 确认"最新地址"的方法：不要猜，直接问用户"现在地址是什么？我帮你对比"

---

## 坑（Pitfalls）

| 坑 | 现象 | 正确处理 |
|----|------|---------|
| 过度设计 | 用户说"看一下"，你写了一堆分析 | 先给检查结果，用户问再深入 |
| 改而不问 | 发现过时内容直接给修改方案 | 先确认是否真的是旧数据，再问要不要改 |
| 漏掉侧边栏 | 只看了主内容区，没检查侧边栏和页脚 | 三层结构检查是必做项 |
| 问错问题 | 问"你现在的地址是什么我帮你改" | 先报告发现的问题，让用户确认→再改 |
| 忘记确认 | 默认所有文案都要更新 | 有些内容（如历史成就）用户可能故意留着 |
