# 新素材整合工作流 v1.0

> 目标：将新提取的销售学素材（OCR文本/PDF/docx/笔记/用户文件）系统性地注入 S1-S5 知识库
> 适用场景：用户发送新文件、cron双周搜索发现新策略、新销售书籍加入
> 工作路径：`/mnt/d/360MoveData/Users/Admin/Desktop/价格&销售&沟通/` + `~/.hermes/skills/productivity/sales-negotiation-customer-service/references/`

---

## 一、素材来源判断

| 来源类型 | 处理方式 | 耗时 |
|---------|---------|------|
| **有文字层PDF** | `pdftotext` 直取 | 秒级 |
| **扫描版PDF** | `pdftoppm` + `tesseract` OCR | 数小时 |
| **docx文件** | `python-docx` 提取 | 秒级 |
| **用户直接提供** | 手动结构化整理 | 按量定 |
| **web搜索发现** | 直接精炼提取关键观点 | 分钟级 |

> OCR提取具体操作见 `references/OCR知识提取_实操指南.md`

## 二、S1-S5 分类映射规则

新素材提取后，逐段/逐条判断归属：

```
S1 客户心理画像
    └─ 客户类型判断、影响力原理、购买信号、心理分析
    └─ 关键词：怕什么、要什么、性格、痛点

S2 价值重塑系统
    └─ 重新定义产品价值、提问技巧、FABG、差异化
    └─ 关键词：SPIN、价值、利益、差异化、升维

S3 价格博弈战术
    └─ 砍价应对、涨价沟通、比价反击、报价策略
    └─ 关键词：价格、砍价、便宜、贵、优惠

S4 难缠客户手册
    └─ 异议处理、暴躁客户、拖延型、怀疑型
    └─ 关键词：异议、难缠、反对、不信任、犹豫

S5 逼单成交引擎
    └─ 促单技巧、成交信号、收尾话术、试服务
    └─ 关键词：成交、签约、逼单、付款、合同
```

**跨界处理**：一个素材可能跨多个S层级 → 参考以下优先级矩阵：

```
             S1  S2  S3  S4  S5
客户说太贵了         ✅      ✅  → S3为主，S4为辅
客户说不信任    ✅          ✅     → S1+S4
客户说考虑        ✅          ✅  → S2+S5
```

## 四、注入弹药库的两个层级

### 层级A：快速注入 → `references/S1-S5_核心弹药库_三书融合版.md`

追加到对应S章节，格式：

```
### [新条目标题]（来源：[来源名]）
[内容]
```

### 层级B：深度整合 → SKILL.md

将核心内容写入SKILL.md的对应S章节，遵循以下指南：

```
S1新增：优先写入 1.0~1.3 之间，标注"融合自XXX"
S2新增：优先写入 2.0~2.2 之间
S3新增：优先写入 3.0~3.1 之间
S4新增：优先写入 4.0~4.1 之间
S5新增：优先写入 5.0~5.1 之间
```

**条件判断**：
- 如果新素材跟已有内容**同类型但更强** → 替换旧内容，旧内容保留在`S1-S5_核心弹药库.md`中
- 如果新素材是**全新视角** → 新增子章节（如 `2.3`）
- 如果新素材**冲突/矛盾** → 标注争议说明，不删除旧内容

## 六、三种整合模式速查

| 模式 | 触发条件 | 动作 | 产物 |
|------|---------|------|------|
| **单条目注入** | 1条新话术/技巧 | 直接追加到S1-S5弹药库+更新S章节 | 1个新条目 |
| **批量注入** | 1本新书/1套完整话术 | 10-50条批量归类+跨S章节分布 | 多个S章节更新 |
| **颠覆性重构** | 全新销售方法论/行业变革 | 新增S章节 / 重写某S层级 | 大版本升级 |
