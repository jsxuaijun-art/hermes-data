# Hermes Agent 内存管理操作指南

## 一、核心认知：内存容量与架构

| 概念 | 类比 | 容量 | 说明 |
|:--|:--|:--:|:--|
| **Memory（记忆条）** | 办公桌台面 | 2,200字符 | 放最常用的工具/规则，满了就写不进去 |
| **Skill（技能）** | 抽屉档案盒 | 不限 | 放操作流程、知识体系，可以挂reference文件 |
| **桌面文件** | 文件柜 | 不限 | Word/Excel文档，路径存记忆里就能找到 |

**关键：** 2200字符是引擎层硬上限，没法扩容。只能通过精打细算和分流到Skill来解决。

---

## 二、内存已满时的处理流程

### 步骤0：先检查是否有对应skill已覆盖

清理前先问自己：**这个内容有没有对应的 skill 已经存了？**

- ✅ 建群话术模板 → `client-group-welcome` skill 已有，memory 里的定稿模板可以直接删，只留一句话指向
- ✅ 灵感捕捉操作流程 → 同样适用，skill 比 memory 更适合存操作步骤
- ✅ 合规账话术/销售话术 → `compliant-accounting` skill 已有，memory 只保留"去这个 skill 里找"

**实操教训（2026.5.25）：** 建群话术定稿模板（~500字）之前同时存在 memory 和 skill 里，白白占了 500 字空间。清理后 memory 占用从 100% 降到 51%。

### 步骤1：评估哪些内容可以删

**优先删除（已完成/一次性的任务型内容）：**
- ✅ 已交付的客户方案（如"南通咖萌架构方案"）
- ✅ 已执行完毕的流程记录（如"GEO冷启动共识"已出Word文档）
- ✅ 一次性的操作指令（如某次调试的临时配置）
- ✅ 已写入skill的重复内容（如"建群话术模板"已存在 `client-group-welcome` skill 中）

**需要保留（日常高频使用的规则型内容）：**
- 🔴 财税回复纪律（每天对话都会引用）
- 🔴 表格生成硬规则（每次回复都触发）
- 🔴 建群话术工作流 + 定稿模板（每天都在用）
- 🔴 时间表述纪律（每次说话前检查时间）
- 🔴 Git同步关键档案（每次拉取推送都需要）

### 步骤2：把长内容迁移到 Skill

超过200字以上的操作流程类内容，最适合从memory迁到skill：

| 场景 | memory里怎么写 | skill里怎么写 |
|:--|:--|:--|
| 灵感捕捉流程 | 全部操作步骤 | 一句话"灵感捕捉 → 走 capture-idea skill" |
| 建群话术定稿模板 | 完整模板 | 一句话"建群话术模板在 client-group-welcome skill" |
| GEO执行步骤 | 全部阶段+细节 | Word文档已存桌面，直接看文件 |
| 客户方案细节 | 完整方案描述 | 放IMA笔记本/桌面文档，记忆里只存路径 |

### 步骤3：清理执行

```text
指令格式（可直接对Hermes说）：
"帮我把XXX从记忆里删掉，腾空间出来"
或
"帮我把XXX（一大段内容）写成skill，记忆里只留一句话"
```

### 步骤4：常见释放量参考

| 内容类型 | 典型占字 | 删1条可释放 |
|:--|:--:|:--:|
| 客户方案详情 | 300-500字 | 够塞2-3条新规则 |
| GEO/执行计划 | 200-300字 | 够塞1-2条新规则 |
| 灵感捕捉操作流程 | 200-300字 | 同上 |
| 建群话术模板（已写在skill里） | 400-500字 | 够塞3条新规则 |

## 三、日常维护建议

1. **每月清理1次** — 回顾记忆条，把已完成任务/一次性内容删掉
2. **优先用Skill存操作流程** — 超过150字的操作步骤，直接写skill比塞内存划算
3. **桌面文件做详情存储** — 模板、清单、方案细节走Word/Excel，路径存内存
4. **周期检查容量** — 发现memory写不进去时（报错"exceeds the limit"），走本流程

## 四、常见问题

**Q：Memory和Skill的区别到底是什么？**
A：Memory = 一句话的规则/提醒（"这家客户的会计叫什么"），Skill = 带操作步骤的方法论（"建群话术怎么生成"）。同样1000字内容，写在memory里占一半空间，写在skill里只占一条链接。

**Q：删了记忆会不会忘事？**
A：已删除的任务型记忆通常是**一次性已完成的**（如某客户方案已交付、某共识已出文档）。如果是还需要的操作流程，会先迁移到skill再删memory。

**Q：我能不能自己清理？**
A：可以，直接对Hermes说"把XXX从记忆里删掉"即可。但建议让Hermes帮你判断哪些能删哪些该保留——它知道每个记忆条的实际使用频率。
