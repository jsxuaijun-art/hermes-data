# GEO（生成式引擎优化）方法论

## 核心理念

> SEO 抢位置，GEO 抢"第一句话"。

当用户把问题交给 AI，TA 不再愿意点十个蓝色链接找答案，而是期待一个"像有经验的朋友那样"的汇总判断。谁让 AI 愿意在第一句话里提到谁，谁就赢。

**GEO 的本质：** 不是 SEO 的替代，而是升维——从"让网页被搜到"到"让品牌被 AI 采纳进答案"。

---

## 一、GEO 三把钥匙

### ① 可引用（Evidence-Ready）
AI 不会空口断言，需要可溯源的证据：
- 权威媒体/机构页面
- 专家与真实用户的稳定口径
- 官方页面的清晰表述
- 客观的对比与指标

**操作：** 在 PR、内容、站点上扔出去的一切，是否便于 AI 链接与引用？

### ② 可理解（Machine-Readable）
不是"多装几个插件"那么简单，是把关键信息语义化：
- Schema.org / JSON-LD 的结构标注
- FAQ 块、参数表、时间与地点、价格
- 适用人群与场景、常见问题与禁忌
- 第三方背书与证据链接

**顺序：** 先信息建模，再做结构化。

### ③ 可采纳（Answer-Ready）
GEO 的终点不是"被抓到"，而是"被采纳进答案里"：
- 提供可拿来就用的判断框架
- 场景 → 标准 → 选项 → 取舍 → 注意事项
- 把"可执行的提纲"写在官网、PR、知乎专栏、行业媒体上

**一句话：** 被看见靠证据，被听懂靠结构，被采纳靠框架。

## 二、DSS 原则（源易信息方法论）

| 维度 | 含义 | 操作要点 |
|------|------|---------|
| **D** - Depth of Semantics（语义深度） | 信息丰富、分析深入、逻辑严密 | 提供有见解、有价值的语料，而非简单罗列 |
| **S** - Supported Data（数据支撑） | 基于可验证的事实、可靠数据 | 具体案例、量化数据、明确证据 |
| **S** - Authority Source（权威信源） | 发布者/平台的专业性和权威性 | 公认的专业机构、行业地位、可信背书 |

### "白帽 GEO"原则
- 拒绝"信息投毒"和"黑帽"技巧
- 真实性为核心
- 真实是品牌在 AI 时代最坚固的护城河

## 三、三级信源模型（趣搜科技）

| 级别 | 类型 | 示例 | AI 权重 |
|------|------|------|---------|
| **T1 - 权威层** | 政府官网、学术机构、主流媒体、企业官网 | 总局官网、Bloomberg、企业官网 | 最高 |
| **T2 - 讨论层** | 社交媒体、论坛、垂类媒体 | 知乎专栏、行业媒体、小红书 | 中等 |
| **T3 - 噪音层** | 无验证数据的营销内容 | 纯软文、无数据支撑的推广文 | 最低（AI 会主动降权） |

> AI 的"反洗稿"能力大幅增强：同一句话如果只在品牌官网上找到，第三方信源中无迹可寻，将被归入低置信度池。AI 甚至能追溯历史版本。

## 四、消费者选购路径变化

传统路径：搜索 → 浏览列表 → 筛选 → 对比 → 加购
**新路径：AI 问答 → 电商/门店 → 加购/下单**

```
用户有模糊需求
    → 向 AI 提问
    → 获得含品牌推荐的整合答案
    → 深度追问、对比细节
    → 带着明确需求转向电商/门店
    → 体验、加购、下单
```

- 80%+ 用户在购买 3C、家电、旅游等产品前通过 AI 辅助决策
- AI 成为连接品牌与消费者的**信任中介**
- 超 8 成用户每天使用 AI 搜索或提问；每天 5 次以上搜索占 24.8%

## 五、315 黑帽 GEO 警示（合规底线）

2026 年 315 晚会曝光了黑帽 GEO 的三大违规操作：

| 违规类型 | 操作手法 | 后果 |
|---------|---------|------|
| **信息污染** | 海量重复内容、批量生成虚假宣传 | 信息疲劳、误导公众 |
| **AI 投毒** | 机器刷互动数据、复制权威内容、虚构用户评价 | 品牌高曝光低信任、破坏公平竞争 |
| **低价竞争** | 低价不承诺效果、虚假排名、夸大效果 | 扰乱行业秩序、影响正确认知 |

**对品牌方的启示：**
- 警惕"快速见效、保证排名"的 GEO 服务商
- 合规交付：优化前审核 → 过程中校验 → 优化后可回溯审计
- 金融、医疗等高风险行业必须加强信源审核

**白帽 GEO 的核心立场：**
> 真实性是品牌在 AI 时代最坚固的护城河。AI 学会了跨源交叉验证，甚至能追溯历史版本。任何"信息投毒"终将被淘汰。

### 6.1 内容优化策略

**用户三层解读：**
1. 用户身份解读（你是谁）
2. 用户使用场景（在什么情境下）
3. AI 搜索意图（想解决什么问题）

**内容搭建：**
- 权威层：政府网站、学术机构、主流媒体、官方文档、企业官网
- 讨论层：社交媒体、论坛、垂类媒体等高质量内容平台
- 平台层：不同 AI 平台的引用偏好不同（豆包侧重抖音，文心侧重百度百科/百家号）

### 6.2 结构化内容 Schema 标记

必须使用的结构化标记类型：
- `FAQ` — 常见问题
- `HowTo` — 操作指南
- `Product` — 产品信息
- `Article` — 文章
- `Organization` — 企业信息
- `LocalBusiness` — 本地商家
- `Review` — 评价/评论

### 6.3 多平台分发策略

| AI 平台 | 内容偏好 | 建议渠道 |
|---------|---------|---------|
| 豆包（字节） | 抖音生态内容 | 抖音、火山引擎 |
| 文心一言（百度） | 百度生态内容 | 百度百科、百家号、百度知道 |
| DeepSeek | 高质量长文 | 知乎、专业媒体 |
| 腾讯元宝 | 微信生态 | 公众号、视频号 |
| 通义千问（阿里） | 电商+技术 | 阿里系平台 |

## 七、品牌知识图谱构建

GEO 的战略层面工作是帮品牌构建系统化知识资产：

### 构建步骤
1. **聚合分散知识：** 收集产品、研发、市场、客服等各部门的知识（产品手册、技术参数、专利、认证报告、白皮书等）
2. **建立实体关系网络：** 品牌 → 产品 → 功效/功能 → 人群 → 工艺/技术，层层关联
3. **结构化输出：** 用 Schema.org 标准标记，形成 AI 可自动抓取的品牌知识库

### 内容闭环
```
品牌官方渠道（官网/公众号）
     ↓
用户实评 + 第三方测评 + 权威认证
     ↓
     互相佐证
     ↓
形成 AI 高信任内容矩阵
```

### 传播策略
- 分析不同平台在 AI 信源中的权重
- 将关键信息布局在 AI 信任的节点（T1 信源优先）
- 监测 AI 信源分布，动态调整分发策略
- 以用户搜索意图为原点逆向规划内容创作

## 十七、GEO 内容输出规范

### 输出格式核心原则

GEO-optimized articles should be published to platforms like Zhihu (知乎), 百家号, 公众号. When the user also needs a clean Word version, follow these rules:

**Clean Word document formatting (user preference):**
- No markdown symbols allowed: remove `>`, `**`, `---`, `- ` bullet markers, `* ` list markers, `#` heading markers
- Use native Word heading styles (Heading 1 for title, Heading 2 for sections)
- Tables must use proper Word table formatting with header row shading (gray #E8E8E8)
- Font: 微软雅黑, 11pt body, 18pt title, 14pt section headers
- Page margins: 2.54cm top/bottom, 3.18cm left/right
- Line spacing: 20pt, paragraph spacing 6pt after
- Numbers format: use "到" or "至" instead of "-" or "~" (e.g., "300到800元" not "300-800元")
- All text in plain Chinese paragraphs, no special characters
- Tables left-aligned, not centered

**Preferred tool:** python-docx library. Sample structure: see `references/clean-word-export.md` for a reusable script template.

**Known pitfalls:**
- Windows file locks: if Word has a .docx file open in Windows, WSL cannot delete or overwrite it (look for `~$` lock files on the Desktop). Save with a new filename or close Word first.
- CJK fonts: always set `w:eastAsia` font name via oxml to ensure Chinese characters render correctly

## 九、GEO vs SEO 对比

| 维度 | SEO | GEO |
|------|-----|-----|
| 目标 | 让网页排到搜索首页 | 让品牌被 AI 采纳进答案 |

| 预算投入 | 成熟市场，竞争激烈 | 早期红利，投入产出比高 |

## 十、GEO 的战略定位

### 营销战略中枢
过去：PR、社媒、SEO/SEM、电商 = 渠道孤岛
现在：GEO 扮演**营销战略中枢**角色：

```
        SEO/SEM
           ↑
 PR ─── GEO ─── 电商运营
           ↓
   社媒/内容
```

- 统一营销口径与内容分发逻辑
- 反向指导内容生产与分发
- 确保不同终端、多模态内容语义一致性

### GEO 服务商核心竞争力

| 维度 | 要求 |
|------|------|
| **技术代差** | 自研模型/算法，逆向工程解析 AI 平台偏好 |
| **商业壁垒** | SaaS 分层订阅、垂直行业深耕、业务板块拓展 |
| **交付信任** | 全链路合规交付体系（可审计、可追溯） |

### 行业生态五方角色

| 角色 | 职责 |
|------|------|
| **AI 大模型平台** | 优化规则、升级算法、强化优质内容采信 |
| **内容平台** | 加强内容审核+优质内容培育 |
| **GEO 服务商** | 建立合规可追溯的服务流程与可量化评估体系 |
| **行业协会** | 制定行业规范、发布行业公约 |
| **品牌方** | 主动布局，选择合规的白帽 GEO 服务商 |

## 十五、财税行业 GEO 实战（融合内容）



### 财税行业高端定位转型的GEO策略（代账→财务顾问/共享财务总监）

适用于已具备专业资质（高级会计师、多年经验、大客户案例），希望从低价代账转型高端财税服务的公司。

**定位换挡：关键词替换表**
```
✗ 代理记账    → ✓ 财务顾问 / 共享财务总监
✗ 公司注册    → ✓ 财务体系搭建 / 内控合规
✗ 报税        → ✓ 税务筹划 / 税务合规咨询
✗ 低价代账    → ✓ 高端财税顾问 / 财务外包
```

**场景卡位：按用户决策阶段布局内容**

| 用户阶段 | 用户搜索意图 | 对应内容标题示例 |
|---------|-------------|-----------------|
| 意识到问题 | 企业财务不规范怎么办 | 企业财务不规范的5个危险信号（自查清单） |
| 寻找方案 | 共享财务总监 vs 全职 | 共享财务总监还是全职财务总监划算 |
| 对比服务商 | 财务顾问服务内容 | 企业财务顾问服务内容全解析 |
