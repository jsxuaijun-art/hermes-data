# 企业初步诊断Skill — 初次洽谈版

> **版本：** 2026年6月 · 苏州盈信企业管理有限公司
> **适用场景：** 首次拜访/首次电话/客户到访（15-20分钟）
> **核心原则：** 不给全貌，给钩子。让客户主动追着你问"那下一步怎么办"

---

## 一、与现有技能的配合

本技能不替代现有深度技能，而是**前端引流层**：

```

│  enterprise-diagnostic（← 你做这个）                                │
│  初次见面的15-20分钟诊断：给3个钩子 + 留1个悬念 ➔ 预约下次深度    │
├─────────────────────────────────────────────────────────────────────┤
│                          ↓ 客户签约/预约                           │
├─────────────────────────────────────────────────────────────────────┤
│  compliant-accounting（已有）                                       │
│  合规账谈单/签约/交付全链路                                         │
├─────────────────────────────────────────────────────────────────────┤
│  corporate-tax-planning（已有）                                     │
│  架构重组/税务筹划深度方案                                           │
├─────────────────────────────────────────────────────────────────────┤
│  equity-architecture-guide（新建）                                  │
│  股权架构全案：公司注册实务/股权激励设计/股东类型进出/七条比例线   │
└─────────────────────────────────────────────────────────────────────┘
```

## 二、双模式设计（关键结构）

| 模式 | 适用 | 耗时 | 信息采集 | 输出 | 目标 |
|------|------|------|---------|------|------|
| **Teaser模式（入口）** | 初次见面/初次电话 | 15-20分钟 | 口头问5-8个问题 | 3个高冲击发现+1个悬念 | 引发兴趣→预约下次 |
| **Deep Dive模式** | 已预约深度诊断/签约客户 | 1-2小时 | 完整采集表 | 12模块完整报告 | 签约/成交 |

**默认入口是Teaser模式。** Teaser跑完后，主动问客户：
> 「老板，刚才说的只是我们初步的判断。如果你有兴趣，我可以给你做一次**完整的企业深度诊断**——12个模块，覆盖股权架构、税务合规、商业模式、未来风险预判等，出一份书面报告。你看要不要安排一下？」

客户同意后，切换Deep Dive模式，使用下方的12模块完整诊断框架输出。

## 三、Teaser模式工作流程（核心内容）

### 第1步：亮身份锚点（30秒）

**不可省略。** 必须先亮身份再问问题。

> 「我是盈信财税的江敏/我的同事。我们先简单介绍一下——苏州做代账的公司几百家，但以代理记账公司名义独立通过**江苏省高级会计师评审**的，只有我们盈信一家。同时我们是**TSC五级涉税专业服务机构**——税务局给涉税服务行业的最高信用评级，438.11分全省前列，全国代理记账行业能达到TSC五级的不到0.5%。」

**话术要点：** 语速平缓、不自夸、像说事实一样自然。

### 第2步：口头采集5-8个关键问题（5-10分钟）

**不是让客户填表，是边聊边问。**

> 「老板，我不耽误你太多时间，问你5个关键问题，我就能给你初步判断一下哪些地方需要注意。」

```
┌───────────────────────────────────────────────────────────────────────────┐
│  □ ① 几个股东？你占多少？                                                 │
│  □ ② 年营收大概什么规模？100万以下？500万？1000万？                      │
│  □ ③ 财务是你自己管还是请了人？代账还是专职会计？                        │
│  □ ④ 你现在感觉交的税多不多？                                            │
│  □ ⑤ 你是做什么行业的？（根据行业追问，见下方定制追问表）                │
│                                                                           │
│  第⑤问专用（先问行业→匹配1个追问，只问1个）：                           │
│  贸易/电商  → 「你供货商能开发票给你吗？」                                │
│  餐饮/食品  → 「你食材采购一个月多少？能拿到发票吗？」                     │
│  工程/装修  → 「你包工头的工资走什么渠道发的？」                          │
│  科技/互联网→ 「你有专利或软著吗？」                                      │
│  制造业    → 「你卖废料的钱走哪里？」                                     │
│  零售/实体  → 「你几个店？每个店独立执照还是统一执照？」                   │
│  医疗/教育  → 「你有行业资质证吗？好办吗？」                              │
│  服务/咨询  → 「你的客户要你开票的多不多？」                              │
│  跨境电商  → 「你做出口走哪个监管方式？9810 还是 9610/0110？退过税吗？」 │
└───────────────────────────────────────────────────────────────────────────┘

**问话技巧：**
- 每个问题先说明"我问这个是为了判断XX" → 避免突兀，显得你在'诊断'不是在'查户口'
  「我先了解一下股权结构，这个关系到后面的控制权风险判断」
  「我问你财务在谁手里，这个决定了我从哪个方向帮你分析」
  「税的问题了解一下，帮你判断有没有节税空间」
- 第⑤问的行业追问：只说那1个追问，不要念全部列表。问完客户回答后，看情况可以补一句延伸
- 客户信息不够或避而不谈 → 换个角度再问：「那大概来说是XX还是XX？」不要硬追

### 第3步：输出3个高冲击发现 + 1个悬念（5-8分钟）

**注意：不是输出12个模块，只输出这4个。**

#### 模块A：税务风险—最痛的地方（必须第一个讲）

基于信息判断，定位1-2个最可能出问题的点：

```python
典型组合（按行业）：
├─ 贸易/电商 → 私户收款 + 进项票不足 + 库存账实不符
├─ 餐饮/零售 → 私户收款 + 社保未交 + 发票管理混乱
├─ 工程/装修 → 成本票严重不足 + 包工头个税 + 挂靠风险
├─ 科技/互联网 → 高新资质不符 + 研发费用归集 + 股权激励税务
├─ 医疗/教育 → 行业专项监管合规 + 收费票据 + 劳务用工
├─ 跨境电商 → 9810海外仓账务 + 出口退税分批申报 + 外汇匹配 + 核定征收
└─ 制造业 → 两套账风险 + 出口退税 + 环保合规成本
```

**输出模板（说人话）：**

> 「老板，就你刚才说的这几个情况，我先讲一个最需要注意的——」
>
> ⚡ **问题：** （根据客户回答动态定位最可能出问题的点）
>   例①（客户说税交得多 + 财务自己管）→ 「你税交得多不一定是因为你赚得多，可能是你的成本没扣除到位，或者你的经营结构可以优化」
>   例②（客户说税交得还行 + 请了代账）→ 「代账只帮你做账报税，他不会帮你做税务规划——这两个是两回事」
>   例③（客户工程行业 + 包工头工资问题）→ 「你包工头的工资如果走现金或私户发，既不能抵成本，还有劳务个税风险」
> ⚡ **风险：** 金税四期已经实现"银税互通"，数据比对能力提升了10倍不止。一个环节出问题，系统会连带排查其他环节。补税+滞纳金+罚款，轻则几十万，重则进去。
> ⚡ **你同行已经踩过的坑：** （举一个真实案例，简短，1-2句话）
> ⚡ **现在该做的事：** 不用马上改，但至少要知道这个风险在。具体方案我们下次坐下来详细聊。

**金句库（背下来，自然说出）：**

| 场景 | 金句 |
|------|------|
| 客户说"我们一直都这么做的" | 「以前查不到不等于以后查不到。金税四期上线后，数据比对能力提升了10倍不止」 |
| 客户说"我朋友也没事" | 「你朋友可能是没查到，查到了你也看不到」 |
| 客户说"你这是在吓唬我" | 「我干了25年，见过太多老板出事前都觉得没事。我是帮你把关的，不是吓你的」 |
| 客户说"那你说怎么办"（🎯 最佳反应） | 「方向有好几个，但要根据你的实际情况来定。你什么时候方便，我们坐下来花一个小时把完整方案给你梳理一遍」 |

#### 模块B：股权/架构风险 — 最容易让老板"啊"的地方（第二个讲）

**如果客户有多个股东/有合伙人计划/营收较大，这个模块必须上。**

> 「刚才你说到XX%持股，这个持股比例从法律角度看有个隐患——」

**输出模板：**

```
⚡ 问题：股权结构隐患
  └─ 67%以下无绝对控制权 → 重大决策无法独立通过
  └─ 50%/50% → 僵局风险，谁都不让谁
  └─ 平均分配（33%/33%/33%）→ 任何两个人联合就能推翻第三人
  └─ 创始人持股过低 + 无一致行动人协议 → 被踢出局风险

⚡ 案例（选一个符合行业的）：
  └─ 真功夫：蔡达标50%/潘宇海50% → 僵局12年 → 蔡达标入狱14年
  └─ 小肥羊：创始团队股权分散 → 被外资并购失去控制权
  └─ 餐饮店50%/50% → 夫妻朋友反目 → 关店清算

⚡ 影响：影响融资、影响决策效率、影响公司长远经营
⚡ 建议：需要调整持股比例或签一致行动人协议。这个不难，但要有针对性方案
```

#### 模块C：商业模式/利润提升点 — 最让老板"咦"的地方（第三个讲）

**注意：基于了解到的信息，点出1个他没意识到的利润增长点。**

> 「另外我注意到，你这个业务模式其实有个利润增长点——」

**输出模板：**

```
⚡ 当前：XX（客户目前的模式）
⚡ 你没注意到的是：XX（专业角度发现的机会）
  └─ 例：你个体户的税率比有限公司低XX%，但你的客户要专票开不出来，丢了很多大单
  └─ 例：你的业务可以拆成XX+XX，各适用不同税率，综合税负降低30-50%
  └─ 例：你的供应商其实可以谈开票，开了票你就能抵税，实际成本反而低了
  └─ 例：你的成本结构里XX项可以合规化处理，利润多留XX%
```

**核心：不是给最终方案，是给他一个"这还能这么操作"的感觉。**

#### 模块D：悬念（必须留）— 预约下回的钩子

**这是整个流程最重要的一步。前面3个模块已经建立了"这个人懂行"的信任，现在是制造"想了解更多"的冲动。**

> 「老板，以上我说的这几点，你可能觉得都是问题，但在我看来——这些都是能解决的事。关键是怎么根据你的实际情况设计最优方案。」
>
> **留悬念：**
> 「其实我今天说的只是一小部分。真正完整的诊断还包括——你的公司架构怎么设计能实现风险隔离、你的税务方案怎么优化能合法节税、你的股权怎么设计能让后面融资不吃亏。这些一两句话说不完，如果你有兴趣，我们约个时间，我给你做一次完整的诊断分析，出一份书面报告给你。」
>
> **如果客户当场要方案（🎯 好信号但不是成交）：**
> 「方向我可以现在就跟你说，但具体方案需要我回去根据你的数据仔细算一下。而且方案涉及好几个选项，不同的选择对你未来几年影响很大，不能草率。你看今天还是明天方便？我到你公司（你来我这边/视频聊）都行。」

## 四、完整话术模板（Teaser实战版）

以下是15-20分钟初次洽谈的标准流程：

### 开场（第1-2分钟）

> 「老板你好，我是盈信财税的XX。我们盈信是做财税服务的，跟市面上一般的代账公司不太一样——我们自己有一支高级会计师带队的团队（苏州唯一以代账公司名义通过高会评审的），同时也是TSC五级涉税机构（税务局最高评级）。所以我们不是简单的帮你做账报税，而是真正帮企业做好规划、控制风险。」
>
> 「今天来的目的也很简单——跟你聊15分钟，我就能对你们公司的情况做一个初步判断，告诉你哪些地方需要注意、哪些地方可以优化。不用填表，你就跟我说就行。」

### 信息采集（第3-10分钟）

边聊边记录要点。用自然对话节奏问。

**开场破冰：**
> 「我先了解一下基本情况——」 （不要问「个体户还是有限公司」，默认有限公司）

**推进节奏（按顺序）：**

```python
# 问法示例（自然对话，不要像读列表）

① 股权情况：
  「你公司几个股东？你占多少？」
  → 「我问这个是判断你的控制权有没有风险」

② 营收规模：
  「年营收大概什么规模？100万以下，还是500万、1000万？」
  → 「这个影响很多——税率、政策、还有你的经营阶段」

③ 财务负责人：
  「财务是你自己管还是请了人？」
  → 「自己管」→ 老板没精力管税务，风险黑洞
  → 「代账管的」→ 代账不做规划，只是帮你报税
  → 「有专职会计」→ 问什么级别的？中级还是初级？

④ 税负感知：
  「你现在感觉交的税多不多？」
  → 无论回答"多"还是"不多"，都有信息差可以展开

⑤ 行业追问：
  「你是做什么行业的？」
  → 客户回答后，匹配行业定制追问（见上方定制追问表），只问1个
  → 客户回答后看情况补一句延伸，不要硬转
```

### 诊断输出（第11-18分钟）

按照第3步的模板，输出3个发现+1个悬念。

**顺序优先级：**
1. 税务风险（最痛、最紧迫）→ 制造危机感
2. 股权/架构风险（如果适用）→ 制造"啊"感
3. 利润提升点 → 制造"咦"感（转负为正，不能全是坏消息）

**过渡话术：**

> 说完风险后：「不过老板你也别担心，我刚才说的这些问题，我们经手的客户都有类似的。关键是怎么解决。」

> 说完利润点后：「这些方向很多老板确实没注意到，但做成了每年能多留几十万在口袋里。」

> 留悬念：「完整的方案需要根据你的数据仔细算，一两句话说不清楚。你看我们约个时间，我这边花1-2小时给你做一次完整的企业诊断？出份书面报告给你。」

## 五、Deep Dive模式（完整诊断）

> **参考源文件：** `references/original-12-module-template.md` 保留了江姐原始12模块提示词全文，供对照或深度参考。
> **公司注册场景：** `references/company-registration-upsell.md` 是客户来注册公司时的专用话术和流程——从注册入口延伸代账/税务/股权服务。
> **出口合规参考：** `references/us-eu-textile-export-certification.md` 是纺织品/婚纱出口美国（CPC/GCC）和欧盟（CE/REACH）的认证知识库——客户涉及出口业务时可快速查阅合规要求与检测机构。

客户在Teaser结束后被勾起兴趣，说「好，你详细讲讲」或主动约时间，切换到此模式。

**模式切换信号：**
- 客户说「那你说怎么办」←🎯 最佳信号
- 客户说「那你给我分析分析」
- 客户主动约下次见面时间
- 你主动追问后客户同意

**前置步骤：信息采集**

在进行12模块输出前，必须先让客户提供完整信息。如果信息不足，禁止硬分析。

使用 corporate-tax-planning 的完整采集清单，比Teaser更深入：

```
一、基础信息         二、股权信息          三、经营情况          四、未来规划
├─ 企业名称          ├─ 股东人数           ├─ 主要产品          ├─ 是否融资
├─ 所在地区          ├─ 持股比例           ├─ 主要客户          ├─ 是否扩大规模
├─ 所属行业          ├─ 股东关系           ├─ 客户来源          ├─ 是否设分公司
├─ 经营范围          ├─ 实际控制人         ├─ 主要竞争对手      ├─ 是否引入合伙人
├─ 成立时间          ├─ 谁负责业务         ├─ 毛利率            ├─ 是否上市
├─ 当前阶段          ├─ 谁负责财务         ├─ 是否私户收款      ├─ 是否有海外业务
├─ 年营业额                                ├─ 是否无票成本支出

## 八、参考话术 — 实战场景模拟

### 场景1：客户是做餐饮的（年营收300万，股东2人各50%，财务自己管）

> **开场（1分钟）：** 「老板你好，我是盈信财税的XX……」
>
> **采集（5分钟）：** 「你公司几个股东？两个。你占多少？50%……好，这个股权比例我等下跟你讲，有个重要的事。年营收大概多少？300万。财务是你自己管还是请了人？自己管？自己管的话税务这一块很容易有遗漏，很多老板不知道。你现在感觉交的税多不多？觉得多是吧——那刚好，我等下跟你讲几个你可能没注意到的方向。你是做餐饮的——你食材采购一个月多少？能拿到发票吗？」
>
> **发现（8分钟）：**
>
> 「老板，我直说——你目前最大的问题不是业务，是资金流的问题。你的营收300万，如果大部分走私户，那你在税务系统上显示的基本就是个'空壳户'。金税四期上线后，银行和税务是联网的，你个人账户每个月固定收款十几二十万，系统自动比对，一旦触发预警，就会查你过去三年的流水。」
>
> 「你知道你同行很多被查的是怎么发现的吗？不是被举报，是大数据自动发现的。查到了，补税+滞纳金+罚款，轻则几十万，重则进去。」
>
> 「但是这个事情不是没办法解决。很多餐饮老板在不知道的情况下把自己暴露在风险里了。如果你愿意，我可以帮你规划一个合规路径——不影响到你目前的经营，但把你和税务局的关系理顺。」
>
> **留悬念（2分钟）：**
>
> 「今天我们聊了这十几分钟，其实还有好几个更关键的方面没深入讲——比如你现在个体户要不要转公司、你未来的利润怎么合理合法地留在口袋、如果你要开分店怎么设计架构不交冤枉税。这些一两句话讲不清楚。你看我们约个时间，我给你做一个完整的诊断？」

### 场景2：客户是做工程的（年营收800万，有限公司，2个股东，成本票严重不足）

> **开场（1分钟）：** （同上）
>
> **采集（5分钟）：** 「几个股东？你跟XX各50%？……那你这个持股比例有个很有意思的问题，等一下我跟你讲。年营收大概多少？800万。财务是你自己管还是请了人？请了代账？代账一般只帮你报税，不会帮你做规划——这个差距很大。你现在感觉交的税多不多？多对吧，我猜可能跟你的成本结构有关。你做工程的——包工头的工资怎么发的？现金还是转账？」
>
> **发现（8分钟）：**
>
> 「先说股权——你俩各50%，从法律上讲这叫'僵局股权结构'。如果你们两个人意见一致那没事，但如果哪一天在经营方向上产生分歧——比如你想扩大他想保守——那就谁都说服不了谁。这种情况我见过太多，最后只能散伙、清算、甚至打官司。」
>
> 「再说税务的问题。你说税交得多，很可能是你的成本没扣除到位。你包工头的工资如果走现金或私户发，既不能抵成本，还有劳务个税风险。而且代账公司只帮你按发票做账，他不会帮你规划——你的800万营收，如果大量成本没票，那你实际利润看着很高，要交的税很多。但你的真实利润其实没那么高，因为钱花出去了但没有票。」
>
> 「你等于是在帮税务局多交钱——你的供应商不开发票少交了税，你替他们交。这不是个别问题，工程行业我见过太多。但解决方案是有的，而且合法。」
>
>
> **留悬念（2分钟）：** （同上）

## 十、常见错误（江姐经验教训）

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 开场没亮身份 | 客户不信任你说的 | 第一句话亮高会+TSC五级 |
| 信息采集像审问 | 客户烦躁/防备 | 每个问题先说原因「我问这个是为了判断XX」 |
| 输出太多模块 | 客户消化不了/觉得你啰嗦 | 只输出3个发现 + 1个悬念 |
| 客户问"怎么办"时直接给方案 | 客户拿到方案就走了 | 引导预约下次，说"要仔细算" |
| 从头到尾讲风险没有正向 | 客户觉得只有问题没有希望 | 第3个发现讲利润增长点（转负为正） |
| 忘记留悬念 | 客户"哦好的谢谢你"就走了 | 最后一定要留预约钩子 |
| 信息不够时硬分析 | 分析错了失去信任 | 坦承"这块信息不够，先问你几个问题" |
