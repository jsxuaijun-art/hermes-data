# WorkBuddy 终端输出风格指南

江姐要求的终端输出格式。任何与江姐的对话都必须遵守。

## 核心原则

1. **ASCII 边框卡片**：用 ┌─┬─┐ 画表格，用 ┌─┐ 画单框卡片
2. **emoji 标题**：每个区块前加一个主题 emoji，如 📋、🎯、✅
3. **加粗强调**：标题行用 `**markdown bold**`
4. **表格对齐**：中文字符占2列宽，计算填充宽度时必须用 CJK 感知算法
5. **禁止废话**：直接交付成品，不加"这是你需要的"等冗余语言

## CJK 表格对齐细则

这是最容易被忽视的地方。犯过一次：右侧边框不成直线。

### 规则

```
每个中文字符 / 全角符号 = 2 个显示列宽
每个英文字符 / 数字 / 半角符号 = 1 个显示列宽
单元格填充 = (列总宽度 - 内容实际显示宽度) * 填充字符
```

### 快速检测

渲染后如果 `┐` 或 `┤` 偏离了上方对应符号的纵列位置，说明填充计算有误。

### 安全做法

对于含中文的列，直接按"每字节=1列"的思维会导致输出偏短（因为中文字符在大多数终端实际占2列）。正确做法：

```
def display_width(s):
    """中文字符算2，英文算1"""
    count = 0
    for c in s:
        if ord(c) > 0x4e00:  # CJK 统一表意文字区
            count += 2
        else:
            count += 1
    return count

# 填充时：
padding = total_width - display_width(content)
```
