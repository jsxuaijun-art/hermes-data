# 拒绝代开发票客户话术

## 场景说明

**客户要求**：委托代开发票（喊同行跑税务局代开免费发票）。
**盈信政策**：不提供代开发票服务。
**核心原因**：一个人的时间是有限的，做账的时间花在开发票上，做账质量就会下降。

---

## 话术模板

### 版本A：直接陈述版（推荐，清晰有力）

```
【公司名称/老板称呼】，我们不开发票的。

原因简单——一个人的时间是有限的，有同行提供免费代开发票，那是人家的服务模式。
我把时间花在开发票上，做账的时间就会又少。

我们的精力放在把账做清楚、申报做仔细上。发票这块，您方如有需要，
建议另外找能代开的同行或者自己去税务局申请代开。
```

### 版本B：委婉引导版（客户关系较近时用）

```
【老板称呼】，发票这事儿我跟您说句实在话——市面上很多同行免费帮开，
但那不是我们的方向。我把人力都用在做账、税务合规、政策梳理上，
开票这个活我们不接。

合作的前提是各自专注。您看发票这块您自己解决，我保证账务这块我们做到位。
```

### 版本C：极简版（群里快速回应）

```
我们不做代开发票的。时间花在开发票上，做账质量就会受影响。
咱们合作的核心价值是做账合规，不是开票方便。
```

---

## 背后逻辑（可解释给客户）

1. **精力分配**：代开发票是纯事务性工作，耗时且无技术含量。同样1小时，花在发票上不如花在账务质量提升上。
2. **专业性定位**：免费代开发票是"跑腿服务"，做高端账务是"专业服务"。二者服务模式不同，不宜混在一起。
3. **风险隔离**：代开发票过程中若出现发票丢失、开错票、交接延误等问题，反而影响客户对账务专业能力的信任。

---

## 拒绝后话术衔接

客户被拒绝后，可以顺势给一个解决方案：

```
不过我可以给您推荐一个专门做代开发票的机构/渠道，
您直接找他们就行，我这边把账务做到位就行。
```

或者引导客户接受电子税务局自助代开（如果客户是自然人/小规模纳税人）。

---

## 注意事项

- 不要贬低同行（"免费代开是垃圾服务"等），只说"服务模式不同"
