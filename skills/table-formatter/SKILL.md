# 终端表格格式化工具

## 风格规范（用户确认的最终版）

- 全细线风格：外框 `┌─┬─┐`，内线 `├─┼─┤`，竖线 `│`
- 每行有横线分隔（网格表）
- 左右竖线对齐（中文字宽双倍计算）
- 单元格内容前后各空1个空格

## 核心函数

```python
def dw(s):
    """计算终端显示宽度：中文=2，英文=1"""
    w = 0
    for c in s:
        if '\u4e00' <= c <= '\u9fff' or '\u3000' <= c <= '\u303f' or '\uff00' <= c <= '\uffef':
            w += 2
        else:
            w += 1
    return w

def pc(s, tw):
    """填充单元格至目标显示宽度"""
    c = f' {s} '
    while dw(c) < tw:
        c += ' '
    return c

def mt(h, r):
    """
    生成全细线网格表格。
    
    ┌──────┬──────────┬──────────────┐
    │ 维度 │ SEO      │ GEO          │
    ├──────┼──────────┼──────────────┤
    │ 目标 │ 上搜索首页│ 进AI第一句话 │
    ├──────┼──────────┼──────────────┤
    │ 核心 │ 算法排名  │ 信任关联     │
    └──────┴──────────┴──────────────┘
    
    h: 表头列表
    r: 数据行列表（每行也是一个列表）
    """
    n = len(h)
    # 计算每列宽度
    cw = []
    for i in range(n):
        m = dw(f' {h[i]} ')
        for row in r:
            x = dw(f' {row[i]} ')
            if x > m: m = x
        cw.append(m)
    
    top = '┌' + '┬'.join(['─' * w for w in cw]) + '┐'
    sep = '├' + '┼'.join(['─' * w for w in cw]) + '┤'
    bot = '└' + '┴'.join(['─' * w for w in cw]) + '┘'
    
    L = [top]
    L.append('│' + '│'.join([pc(h[i], cw[i]) for i in range(n)]) + '│')
    L.append(sep)
    
    for idx, row in enumerate(r):
        L.append('│' + '│'.join([pc(str(row[i]), cw[i]) for i in range(n)]) + '│')
        if idx < len(r) - 1:
            L.append(sep)
    
    L.append(bot)
    return '\n'.join(L)
```

## 注意事项

1. **中文字宽**：CJK统一表意文字（\u4e00-\u9fff）、CJK符号（\u3000-\u303f）、全角ASCII（\uff00-\uffef）按2宽度计算
2. **禁止输出 markdown 表格**：`| --- | --- |` 这种格式绝对不要用
3. **边生成边验证**：每次先调 mt() 生成字符串，确认对齐无误后再贴到回复中
4. **最佳列数**：2~6列，超过6列终端可能折行
5. **表头和数据行写法**：数据行不要用 `┃`（厚竖线），统一用 `│`（细竖线）

## 踩坑记录 / Pitfalls

### 终端渲染宽度不一致
- `dw()` 函数用 Unicode 范围（\u4e00-\u9fff 等）判断中文字符宽度为2，但不同终端/字体渲染同一字符的方式不同
- 特别是全角括号 `（` `）`（U+FF08/U+FF09）在部分终端可能渲染为宽度1或不规则宽度
- 用户报告右侧竖线偏移时，执行以下诊断：
  - 打印最宽行每个字符的 ord() 和 dw() 值
  - 用 `+ - |` ASCII 版临时替代，确认是宽度问题还是渲染问题
  - 如果只有特定终端偏，保持 Unicode 版本，不降级
- 已获得用户（江敏/manager）明确确认：全细线网格格式为最终版，不再使用 Markdown 虚线表
