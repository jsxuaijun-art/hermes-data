# Agent 类型与工具能力

## 可用 Agent 类型

通过 Task 工具的 `subagent_type` 参数指定 agent 类型。不同类型拥有不同的工具集。

### general-purpose（通用型）

**工具**：全部工具（Read, Edit, Write, Bash, Glob, Grep, Task 等）
**适用**：需要读写文件、执行命令的实现类任务
**典型角色**：前端实现、后端实现、测试编写、文档更新、重构执行

### Explore（探索型）

**工具**：只读工具（Read, Glob, Grep, LS, WebFetch, WebSearch 等），**不能** Edit/Write/Bash
**适用**：代码搜索、架构分析、代码审查、信息收集
**典型角色**：安全审查、性能分析、代码审计、假设调查、研究员

### Plan（规划型）

**工具**：只读工具，**不能** Edit/Write
**适用**：架构设计、方案规划、依赖分析
**典型角色**：架构师、规划师、方案设计

### Bash（命令型）

**工具**：仅 Bash
**适用**：纯命令行操作（git、npm、docker 等）
**典型角色**：CI/CD 操作、环境配置、构建部署

## Agent 类型选择指南

| 任务性质 | 推荐类型 | 原因 |
|---------|---------|------|
| 写代码/改文件 | general-purpose | 需要 Edit/Write 工具 |
| 代码审查/分析 | Explore | 只需读取，更安全 |
| 架构设计 | Plan | 只需分析，不改代码 |
| 运行测试/构建 | general-purpose | 需要 Bash 执行命令 |
| 搜索/调研 | Explore | 只需搜索和阅读 |

## 模型选择

通过 Task 工具的 `model` 参数指定模型：

| 模型 | 适用场景 | 成本 |
|------|---------|------|
| opus | 复杂推理、架构设计、关键决策 | 最高 |
| sonnet | 大多数实现任务、代码审查 | 中等 |
| haiku | 简单搜索、格式化、快速查询 | 最低 |

**建议**：
- 领导和架构师用 opus 或 sonnet
- 实现者用 sonnet
- 简单搜索/格式化用 haiku

## 权限模式

通过 Task 工具的 `mode` 参数控制：

| 模式 | 说明 | 适用 |
|------|------|------|
| default | 需要用户确认敏感操作 | 大多数场景 |
| plan | 只能规划，需审批后执行 | 架构师、需要审批的角色 |
| bypassPermissions | 跳过权限确认 | 信任的自动化任务 |

## Teammate 提示词编写要点

1. **明确角色**：开头说明角色和职责
2. **限定范围**：指定可操作的文件/目录
3. **输出格式**：要求结构化输出（Markdown 报告、JSON 等）
4. **协作规则**：说明何时需要与其他 teammate 沟通
5. **完成标准**：明确什么算"完成"

**提示词模板**：
```
角色：{role_name}
职责：{responsibilities}
文件范围：只修改 {directories}，不要触碰 {excluded_dirs}
输出要求：完成后提交结构化报告到任务列表
协作：发现影响其他模块的变更时，通过消息通知相关 teammate
完成标准：{completion_criteria}
```
