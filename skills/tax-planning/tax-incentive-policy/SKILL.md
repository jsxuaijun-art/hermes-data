# 中国税收优惠政策研究与客户咨询方法论

## 概览

本技能覆盖税收优惠政策的**检索→核验→测算→呈现**全流程。基于苏州盈信财税高级会计师江敏实战经验。

**核心原则：**
1. **信息来源优先顺序：** 国家税务总局官网 > 省级税务局 > 律师事务所解读 > 一般资讯网站
2. **每项政策必须标注文件号**（如财税〔2023〕8号、国税〔2016〕52号）
3. **政策有效期必须明确**（执行至哪年哪月哪日）
4. **效益测算必须附带假设条件**，不可把估算数当确定数
5. **"听起来好≠合规"** — 享受条件逐条核验，缺一不可

## 第一阶段：政策检索方法

### 搜索策略

```text
① 明确搜索方向 → ② 用关键词组合搜索 → ③ 提取官方文件 → ④ 交叉验证
```

**推荐搜索来源（优先级从高到低）：**

| 来源 | 网址 | 内容 |
|------|------|------|
| 国家税务总局法规库 | fgk.chinatax.gov.cn | 财政部+税务总局联合发文，最权威 |
| 各地税务局官网 | xx.chinatax.gov.cn | 地方执行口径、申报指南 |
| 税屋网 | shui5.cn | 归集较全，但可能缺少最新政策 |
| 律师事务所解读 | (如惠城、金杜等) | 解读详细，但需要核实法条原文 |

### 关键词组合技巧

- 政策名 + "文件号" → 找原文（如"残疾人就业 增值税 即征即退 文件"）
- 政策名 + "享受条件" → 找条件（如"加计扣除 享受条件"）
- 政策名 + "延续" → 查有效期（如"残保金 优惠政策 延续 公告"）
- 政策名 + "申报" → 查操作（如"残疾人就业保障金 申报"）

### ⚠️ 政策检索陷阱

1. **旧政策被新政策替代了却没发现**
   - 多个相似名称的政策并行 → 必须确认哪个是最新的
   - 搜索时加年份（如"2024 2025"）过滤旧信息
2. **地方口径 ≠ 全国口径**
   - 深圳/上海的地方通告 ≠ 全国通用
   - 引用地方口径时务必注明"XX省/市规定"
3. **律师网站的政策可能过时**
   - 网页显示的"更新时间"可能是页面修改时间，非政策实效
   - 交叉验证：找官方文件核对有效期限
4. **分档优惠容易被忽略叠加**
   - 同一件事可能有多个优惠政策 → 需要判断能否叠加享受
   - 如：小微企业所得税减免 + 高新优惠 → 选择最优，不能叠加

## 第二阶段：政策核验流程

### 每项政策的核验清单

```
□ 政策全称 + 文件号（财政/税务/人社/残联联合发文）
□ 发布机关（财政部？税务总局？地方？）
□ 适用主体（企业/个体工商户/事业单位/机关？）
□ 享受条件（逐条列出，缺一不可）
□ 有效期限（从哪年到哪年，是否已延续）
□ 申报方式（自行申报/备案/审批？）
□ 叠加规则（能否与其他优惠同时享受？）
```

### 残疾人就业相关政策的核验要点（典型）

```
□ 残疾人证类型：残疾人证？残疾军人证（1-8级）？
□ 劳动合同：签订1年以上？并在岗工作？
□ 社保：按月足额缴纳（养老+医疗+失业+工伤+生育）？
□ 工资：通过银行发放？不低于当地最低工资？
□ 设施：具备安置残疾人上岗的基本设施？
```

## 第三阶段：效益测算框架

### 测算规则

1. **用具体数字说话** — "省税约XX万元/年" 比 "可以省不少税" 有说服力
2. **分项列出** — 残保金省多少、增值税省多少、所得税省多少，分别列
3. **标注假设条件** — 每项测算后面标注"按年平均工资XX万估算"等
4. **风险提示并列** — 省税收益和可能的风险并存

### 标准测算模板

```text
┌─────────────────────────────────────────────────────────┐
│  【XX优惠政策】效益测算                                 │
│                                                        │
│  假设条件：                                            │
│  • 公司人数：XX人                                      │
│  • 安置残疾人：XX人（占比XX%）                         │
│  • 残疾人平均年薪：XX万/年                            │
│  • 当地月最低工资：XX元                                │
│  • 当地年平均工资：XX万                                │
│                                                        │
│  项目                     节省金额       计算依据     │
│  ─────────────────────────────────────────────────     │
│  ① 免缴残保金             XX万/年     公式见下      │
│  ② 增值税即征即退         XX万/年     文件号        │
│  ③ 企业所得税加计扣除     XX万/年     文件号        │
│  ④ 城镇土地使用税减免     XX万/年     文件号        │
│  ─────────────────────────────────────────────────     │
│  合计                    ≈XX万/年                     │
│                                                        │
│  【风险提示】                                          │
│  • 上述测算基于假设条件，实际以税务机关核定为准      │
│  • 享受条件需逐条满足，缺一不可                      │
│  • 各地执行口径可能有差异，建议咨询当地税务局        │
└─────────────────────────────────────────────────────────┘
```

## 第四阶段：客户沟通话术

### 初问型客户（"我就问问有什么优惠"）

```
"老板，优惠政策确实不少，但能不能享受，关键看你们公司的实际情况。
我帮您看看：您公司多少人？有残疾人吗？交的是小微企业还是一般纳税人？
营业额大概多少？—— 几分钟就能判断几条路。"

（✅ 引发客户提供信息，而不是直接上课）
```

### 想省钱型客户（"那残疾人怎么招？"）

```
"招残疾人不难，残联有就业服务平台可以对接。
难的是合规——合同、社保、工资、银行流水，四条线缺一不可。
差一条，不但优惠享受不到，还可能被认定为假用工，倒查补税。
我帮您列一个合规清单，您对照着准备。"
```

### 怀疑型客户（"别给我画饼，说风险"）

```
"您说得对，任何优惠政策都有两面性。
享受残疾人优惠最大的风险是：残联和税务会有联动检查。
社保不交 ✓系统预警→ 税务核查→ 补税+滞纳金。
所以我一贯的原则是：先把合规底子打好，再谈省钱。
这套方案，合规是前提，省钱是结果。"
```

### ⚠️ 话术注意事项

- ❌ 不要一上来就长篇大论政策原文（客户不想听课）
- ✅ 先问情况，再定向给出1-2个最有可能的方案
- ❌ 不要把测算数字说死（"肯定能省XX万"→ 风险提示跟上）
- ✅ 用案例说话（"之前有个XX行业的客户，做了这个方案后……"）
- ❌ 不要替客户做决定（"建议您用这个方案"→ 给出选项，让客户选）

## 第五阶段：交付标准

### 输出格式

- **桌面 .docx**（默认交付格式，纯Python生成）
- **终端表格**（配合 table-formatter 或 workbuddy-output 技能，不用Markdown表格）
- **政策参考文件** → 保存到 `references/` 目录，下次同类查询可以直接引用

### 报告结构

```text
一、政策概述（文件号+核心内容）
二、适用条件（逐条核验清单）
三、效益测算（分项列出+合计+假设条件）
四、操作路径（申报步骤+时间节点）
五、风险提示（每项政策的风险点+防范措施）
六、常见问题FAQ（客户可能追问的3-5个问题）

末尾必须加：
「⚡ 以上信息基于公开政策整理，实际执行以当地税务机关口径为准」
```
