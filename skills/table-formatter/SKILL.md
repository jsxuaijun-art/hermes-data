## ⛔ MANDATORY - load this skill before ANY table output

## 风格规范（用户确认的最终版）

- 全细线风格：外框 `┌─┬─┐`，内线 `├─┼─┤`，竖线 `│`
- 每行有横线分隔（网格表）
- 左右竖线对齐（使用 wcwidth 库精确计算字符宽度）
- 单元格内容前后各空1个空格

## 核心函数

**依赖：** `pip install wcwidth`（必须，否则表格 emoji 行会偏移）

```python
from wcwidth import wcswidth

def dw(s):
    """计算终端显示宽度：使用 wcwidth 库，准确处理 emoji、CJK 等"""
    return wcswidth(s)

def pc(s, tw):
    """填充单元格至目标显示宽度"""
    c = f' {s} '
    while dw(c) < tw:
        c += ' '
    return c

def mt(h, r):
    """
    生成全细线网格表格。
    
    ┌──────┬──────────┬──────────────┐
    │ 维度 │ SEO      │ GEO          │
    ├──────┼──────────┼──────────────┤
    │ 目标 │ 上搜索首页│ 进AI第一句话 │
    ├──────┼──────────┼──────────────┤
    │ 核心 │ 算法排名  │ 信任关联     │
    └──────┴──────────┴──────────────┘
    
    h: 表头列表
    r: 数据行列表（每行也是一个列表）
    """
    n = len(h)
    # 计算每列宽度
    cw = []
    for i in range(n):
        m = dw(f' {h[i]} ')
        for row in r:
            x = dw(f' {row[i]} ')
            if x > m: m = x
        cw.append(m)
    
    top = '┌' + '┬'.join(['─' * w for w in cw]) + '┐'
    sep = '├' + '┼'.join(['─' * w for w in cw]) + '┤'
    bot = '└' + '┴'.join(['─' * w for w in cw]) + '┘'
    
    L = [top]
    L.append('│' + '│'.join([pc(h[i], cw[i]) for i in range(n)]) + '│')
    L.append(sep)
    
    for idx, row in enumerate(r):
        L.append('│' + '│'.join([pc(str(row[i]), cw[i]) for i in range(n)]) + '│')
        if idx < len(r) - 1:
            L.append(sep)
    
    L.append(bot)
    return '\n'.join(L)
```

## 注意事项

1. **字符宽度**：使用 `wcwidth` 库（`wcswidth()`）计算显示宽度，支持 CJK、emoji（如 ✅ 👉）、全角符号等
2. **禁止输出 markdown 表格**：`| --- | --- |` 这种格式绝对不要用
3. **边生成边验证**：每次先调 mt() 生成字符串，确认对齐无误后再贴到回复中
4. **最佳列数**：2~6列，超过6列终端可能折行
5. **表头和数据行写法**：数据行不要用 `┃`（厚竖线），统一用 `│`（细竖线）

## 踩坑记录 / Pitfalls

### emoji 宽度问题（已修复）
- `✅`（U+2705）等 emoji 在终端中宽度为 2，但旧版 `dw()` 函数仅识别 CJK 范围，按宽度 1 计算
- 导致含 emoji 的行右侧竖线偏移 1 字符
- **修复方案**：改用 `wcwidth` 库的 `wcswidth()`，该库正确覆盖 emoji、CJK、全角半角等所有 Unicode 范围
- 安装依赖：`pip install wcwidth`
